# Q4：提出一种更“公平/更兴奋”的每周合成规则（DAWS）

目标：给出一套**可在电视节目中解释**、可逐周执行、并能在历史数据上被验证的“评委评分 + 粉丝投票”合成系统，使其在“公平性（尊重技术）”与“兴奋度（尊重观众）”之间取得可量化的折衷。

## 1. 与 Q1/Q2/Q3 的衔接（为什么这不是拍脑袋）

### Q1：提供粉丝投票的统计输入（不是“真值”）
Q1 的 MCMC 反推结果给出的是粉丝投票的**后验分布**（含不确定性），可作为仿真与比较规则的输入。我们不把它当作 ground truth，而把它当作“最贴近历史的一致性估计”。

在 Q4 中，我们明确避免“循环论证”：
- Q1 的后验样本表示的是“在与历史淘汰结果相容的前提下，粉丝投票可能长什么样”的一簇**可能世界（posterior ensemble）**；
- Q4 评估 DAWS 时，不是用这组样本“证明历史是对的”，而是问：**在这些合理的可能世界中，新规则的表现是否稳健（对样本扰动不敏感）**。

稳健性写法（思路层面）：
- 在 Q2 的引擎里，对同一规则用不同子样本/放大噪声的方式重跑一遍，看公平性/兴奋度指标的变化是否小（例如多次重复采样、或对粉丝份额 $F_i$ 叠加小幅扰动）；
- 若 DAWS 的推荐参数在这些“扰动世界”下给出的帕累托前沿/拐点位置基本不变，就可以在论文里强调“结论对具体后验样本的依赖度有限，而是结构性的”。

### Q3：解释为什么需要“分阶段”
如果 Q3 显示：
- 早期阶段非技术因素（行业、名气、搭档效应等）对结果影响更大；
- 后期评委评分对最终名次的解释力更强；
则说明“粉丝逻辑”和“评委逻辑”在赛季不同阶段存在结构性差异。分阶段权重不是常识，而是由 Q3 的差异性证据支撑。

### Q2：用同一套反事实仿真引擎比较规则优劣
Q2 的模拟器可以把“规则”作为输入，并输出一组可比较的指标（例如翻转率、技术冤死风险等）。将 Q4 的新规则接入 Q2，就能在**同一评估框架**下与 Rank/Percent、以及“最后两名评委救人”等机制对比。

## 2. 我们提出的规则：DAWS（Dynamic Adaptive Weighting System）

DAWS 是一个**两阶段、可解释**的合成系统：常规赛强调观众参与，后期强调技术公平。

### 2.1 每周合成分数（核心公式）
对当周仍在赛的 $i$ 号选手：

1) 评委侧归一化：
$$J_i=\frac{S_i}{\sum_{k\in\mathcal{A}} S_k}$$
其中 $S_i$ 为该周评委总分，$\mathcal{A}$ 为当周仍在赛的选手集合。

2) 粉丝侧归一化（来自 Q1 的估计投票）：
$$F_i=\frac{V_i}{\sum_{k\in\mathcal{A}} V_k}$$
其中 $V_i$ 为该周粉丝投票数（未知量，由 Q1 产生后验样本）。

3) 合成得分：
$$C_i=w_t\,F_i+(1-w_t)\,J_i$$
其中 $w_t\in[0,1]$ 是“该周粉丝权重”。当周淘汰 $C_i$ 最低者（或进入后述的“救人”环节）。

### 2.2 两阶段权重（可解释、可营销）
我们将赛季划分为两段：
- **常规赛（Regular Season）**：$w_t=w_{early}$（更重观众参与）
- **季后赛（Playoffs）**：$w_t=w_{late}$（更重技术公平）

切换点用“剩余人数阈值”定义：当剩余人数 $\le N_{cut}$ 时进入季后赛。

约束建议：$w_{early}\ge w_{late}$（前期粉丝权重不低于后期），以保持“先热后稳”的叙事一致性。

### 2.3 完整每周流程与赛制细则（含 Bottom-2 / Save / 并列 / 双淘汰）

**基础流程（所有方案共用）：**
1) 当周仍在赛选手集合为 $\mathcal{A}$，计算每人的 $J_i, F_i, C_i$；
2) 按 $C_i$ 从低到高排序，得到当周“综合表现”名次；
3) 根据赛季实际赛制，决定当周应淘汰的人数 $K$：
	 - 正常单淘汰周：$K=1$；
	 - 无淘汰周：$K=0$（仍可公布排名/Bottom-2 作为“悬念”）；
	 - 双淘汰周：$K=2$（或由清洗数据中的周类型字段给定）。

**并列与平局处理：**
- 若出现 $C_i$ 完全相同的并列情况，先用评委总分 $S_i$ 作为第一层 tie-break（技术优先）；
- 若 $S_i$ 仍相同，则回溯当周粉丝份额 $F_i$（尊重大众意愿）；
- 若三者仍相同（极小概率），用选手姓名字典序打破平局，保证规则可复现、可编程。

在此基础上，我们提出两套可对比的 DAWS 版本：

**方案 A：DAWS-Direct（无 Save）**
- 直接按 $C_i$ 排名，淘汰当周综合得分最低的 $K$ 人；
- 适合作为“规则本体”的基准版本，易于解释、逻辑简单。

**方案 B：DAWS-Save（带 Bottom-2 + 评委救人）**
- 第一步：按 $C_i$ 选出当周综合得分最差的 $K+1$ 人，形成 Bottom 区（单淘汰时为 Bottom-2）；
- 第二步：在 Bottom 区中，评委投票/按 $S_i$ 决定“救谁”：
	- 单淘汰周：在 Bottom-2 中，淘汰 $S_i$ 更低者；
	- 双淘汰周：在 Bottom-3 中，优先保留 $S_i$ 最高的一位，其余两人淘汰；
- 若评委票数平局，可退回到粉丝份额 $F_i$ / 名字字典序作为 tie-break。

在 Q2 引擎中，这两套方案都可以作为“宇宙”之一接入，与现有的 Rank-Direct、Rank-Save、Percent-Direct、Percent-Save 平行比较：
- 直接在 `method` 维度上增加 `DAWS-Direct` 与 `DAWS-Save`，对同一批后验样本跑一遍；
- 用同一组指标（reversal_rate、tech_vulnerability、popularity_vulnerability 以及 rank_vs_percent 的分歧方向）来比较 DAWS 与历史制度；
- 同时在决赛周模块中，把 DAWS 作为另一套“决赛合成规则”，输出冠军/名次分布的变化。

## 3. 规则参数如何确定（去掉拍脑袋与捏造数值）

决策变量：$\theta=(w_{early},\,w_{late},\,N_{cut},\,\text{是否启用 Save})$。

做法：在合理网格/范围内对 $\theta$ 做搜索，并用 Q2 在历史赛季上进行蒙特卡洛评估：
- 每次仿真从 Q1 的后验中抽取一组 $(V_i)$ 作为“可能的粉丝投票世界”；
- 在该世界下按 DAWS 规则按每一周推演；
- 统计指标并在赛季间聚合。

为了避免“针对个别争议赛季调参数”的过拟合，可在思路上采用**留一赛季交叉验证（leave-one-season-out）**：
- 在 $S-1$ 个赛季上搜索/评估一组候选参数，选择在这 $S-1$ 个赛季上表现最稳定、最接近帕累托拐点的一组；
- 再在被留出的那个赛季上检验这组参数的公平性/兴奋度指标是否仍然合理；
- 换不同的“被留出赛季”重复这个过程，用文字说明：DAWS 的推荐参数是在多数赛季上都表现稳健的，而不是“为 S27 单独调出来”的结果。

输出形式建议：
- 画出公平性—兴奋度的帕累托前沿（或热力图）；
- 选择“折衷点”（例如帕累托前沿的拐点）作为推荐参数，而不是直接写死某个比例。

## 4. 评估指标（建议与 Q2 保持一致口径）

为避免“兴奋度”被误写成单一方向，我们建议至少同时报告两类指标：

1) 公平性（技术保护）
- 技术冤死风险（tech_vulnerability）：评委总分最高者（或前几名）在某周被淘汰的概率；在 Q2 已通过 `tech_vulnerability` 指标实现。
- 评委-粉丝对齐度（可选）：例如冠军在评委累计分上的分位数、赛季内高评委分选手整体生存情况，可用 Q2 的决赛分布与 survival/tracker 输出辅助说明。

2) 兴奋度（观众参与感）
- 结果可翻转性：粉丝投票波动对淘汰结果的敏感程度，可以用**机制间的分歧率**和 Q2 中的 `disagreement_rate` 指标来度量（例如 Rank vs Percent、DAWS vs Percent）；也可以在 DAWS 内部对 $F_i$ 做小扰动，看淘汰结果变动的频率。
- 观众有效性（popularity_vulnerability）：人气领先者被淘汰概率不能过高，这在 Q2 中已有 `popularity_vulnerability` 指标，可直接用于比较 DAWS 与历史方法在“保护人气王”上的差异。

此外，Q2 已经输出了 `reversal_rate`（与历史真实淘汰的一致性）和决赛冠军/名次变化等指标，Q4 可以直接引用这些量：
- 把历史 Rank / Percent 与 DAWS 放在同一张“公平性-兴奋度”平面或热力图中，对比其在 `tech_vulnerability`、`popularity_vulnerability`、`reversal_rate` 三个指标上的相对位置；
- 对题目点名的争议选手（Jerry Rice, Billy Ray Cyrus, Bristol Palin, Bobby Bones），结合 Q2 的 tracker 与 finale 输出，单独比较在 Rank / Percent / DAWS（含/不含 Save）下的“被淘汰风险曲线”“期望淘汰周”“决赛名次/夺冠概率”差异，用于支撑文字讨论。

最终推荐以“同时降低技术冤死、但保持适度可翻转性”为目标，而不是单纯追求某一个指标极小/极大。

## 5. 优缺点（客观、可答辩）

优点：
- **可解释**：两阶段叙事（常规赛/季后赛）比连续非线性权重更适合电视转播。
- **数据驱动**：参数通过 Q1+Q2 在历史赛季上搜索得到，避免主观定权。
- **可对比**：能在同一引擎下与 Rank/Percent、以及 Save 机制做反事实比较。

潜在风险与补救：
- **切换点“断崖效应”**：可通过“提前公告”“切换周加保护条款（如 Save）”减弱。
- **对评委偏差的依赖**：若 Q3 检测到评委系统性偏差，需谨慎提高后期评委权重，或在后期引入更多评委/去偏处理。
