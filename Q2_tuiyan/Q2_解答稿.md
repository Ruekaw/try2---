# C题 Q2：赛制机制对比（Rank vs Percent）与“评委救人”环节的影响

本节回答题目第二小问：
1) 在同一周、同一参赛集合下，对比 **Rank 法** 与 **Percent 法** 的淘汰/排名差异，并判断哪种更偏向粉丝投票；
2) 进一步加入“先确定 Bottom-2，再由评委从 Bottom-2 中淘汰一人”（下称 **Save** 机制），分析其对结果与争议选手的影响。

> 重要声明：本 Q2 采用 **逐周反事实（week-level counterfactual）**：固定“历史该周参赛集合”，在同一周里比较不同赛制的决策差异；不模拟反事实淘汰后票迁移与后续周参赛集合的动态反馈。因此，结论解释为“该周赛制倾向/稳定性差异”，而不是严格预测“整季冠军一定如何变化”。

---

## 1. 输入与实验设置（Q1→Q2）

### 1.1 输入数据
- Q1 后验粉丝投票份额样本：每周 10,000 组粉丝份额向量 $F_s$（各选手份额和为 1）。
- 清洗后的评委数据：当周参赛者名单与评委总分 $J$，以及 Percent 制下的当周评委份额 $J^{share}$。

### 1.2 核心周筛选（与题面“单淘汰机制”对齐）
核心统计只覆盖：
- **单淘汰周**（排除双淘汰）
- **非无淘汰周**
- **非决赛周**（决赛周单独做“冠军/名次”专题）
- 该周参赛者有效且未被标记为不可推断

覆盖范围（来自输出统计）：
- 核心周共 788 个 (season, week)
- 被跳过周 106 个，其中：
  - 74 个因为 Q1 未导出该周 NPZ（原因分类里主要落在“无淘汰周/决赛周”）
  - 32 个为双淘汰周
- 第 26 赛季在“核心周”中缺失（其非决赛周均为双淘汰而被过滤）；但仍可在决赛专题中比较名次分布。

---

## 2. 四个机制宇宙（同一周、同一批后验样本）

设当周参赛人数为 $N$，粉丝份额样本矩阵 $F\in\mathbb{R}^{S\times N}$（$S=10000$）。

### 2.1 Rank – Direct
- JudgeRank：对 $J$ 从高到低排名（1 为最好）
- FanRank：对每个样本 $F_s$ 从高到低排名
- 综合名次：$SumRank = Rank(J)+Rank(F_s)$
- 淘汰者：$\arg\max_i SumRank_i$（名次和最大=最差）

### 2.2 Rank – Save
- 先按 Rank–Direct 得到 $SumRank$，取 Bottom-2
- 淘汰 Bottom-2 中 **评委总分 $J$ 更低**者（平局用粉丝份额、再用名字字典序做可复现裁决）

### 2.3 Percent – Direct
- 当周综合份额：$Total = J^{share}+F_s$
- 淘汰者：$\arg\min_i Total_i$

### 2.4 Percent – Save
- 先按 Percent–Direct 得到 Bottom-2
- 淘汰 Bottom-2 中 **评委总分 $J$ 更低**者（平局规则同上）

---

## 3. 指标设计：怎么量化“差异有多大、偏向哪边”

### 3.1 Rank vs Percent：淘汰分歧率与方向性
- 分歧率：
$$D_{elim}=P(E_{rank}\neq E_{percent})$$
- 方向性：在发生分歧的样本中，用“被淘汰者在当周粉丝端/评委端的 badness quantile（0=最好，1=最差）”比较两种机制。
  - 若 Rank 淘汰者在评委端更“差”，则 Rank 更 **judge-friendly**（更保护技术）。
  - 若 Rank 淘汰者在粉丝端更“差”，则 Rank 更 **fan-friendly**（更保护人气）。

### 3.2 Save 机制影响：技术冤死率与人气冤死率
- 技术冤死率（tech vulnerability）：当周评委第一名被淘汰的概率
- 人气冤死率（popularity vulnerability）：当周粉丝第一名被淘汰的概率

> 这两项主要用于解释机制倾向与“争议”的制度来源；数值越小，代表该维度越安全。

### 3.3 决赛专题：冠军是否改变
在决赛周固定参赛集合下，对比 Rank vs Percent 的冠军：
$$D_{winner}=P(winner_{rank}\neq winner_{percent})$$

### 3.4 个案追踪：逐周淘汰风险与“走多远”代理
对题目点名争议选手，输出逐周 $P(Elim=c)$ 曲线，并用 hazard 近似汇总：
- modeled weeks 内全程不淘汰概率（survive_prob_through_modeled_weeks）
- 期望出局周（expected_exit_week）

## 4. 宏观结论 1：Rank vs Percent 差异很大，且 Rank 更偏评委

总体（跨所有可用核心周与 10,000 次后验样本）结论：
- **淘汰分歧率**：$D_{elim}=0.3689$。
  - 解释：在约 36.9% 的后验样本中，Rank 与 Percent 会淘汰不同的人；这意味着“选哪种合成规则”在相当多的周会直接改变淘汰结果。
- **分歧方向**（仅在分歧样本中）：
  - Rank 更 fan-friendly 的比例：0.0291
  - Rank 更 judge-friendly 的比例：0.9658

据此可以得出强结论：
- 当 Rank 与 Percent 发生冲突时，**Rank 几乎总是更“保护评委意见”**；换言之，Percent 在冲突时更容易体现粉丝端的力量（更可能淘汰“评委更喜欢但粉丝更弱/综合份额更低”的人）。

赛季层面的差异并不均匀：
- 分歧率最高的赛季：S34（0.521）、S24（0.480）、S20（0.473）、S28（0.466）、S33（0.466）等。
- 分歧率最低的赛季：S16（0.222）、S10（0.226）、S3（0.251）、S12（0.256）等。

解释（写作建议）：
- 当“评委分差距很大但粉丝份额分散/极化”时，两种机制更容易给出不同淘汰者；
- Rank 的“排名化”会压缩极端差距，使评委端的相对位置（而不是绝对差）更稳定地决定 Bottom 区域；Percent 则保留了份额差距的强度，更容易被粉丝端的大幅波动拉动。

---

## 5. 宏观结论 2：Save 机制显著降低“技术冤死”，但会系统性压制争议选手

在核心周总体统计中：
- Percent–Direct 的技术冤死率为 0.0623（明显高于 Rank 系列）
- 加入 Save 后，Percent–Save 的技术冤死率降到 0.0052

这说明：
- **Save 机制本质上是“技术兜底”**：只要技术强者不落入 Bottom-2，或即便落入 Bottom-2 但技术更强，评委可将其救下。
- 该机制会压缩“人气逆袭”的空间：凡是依赖粉丝份额把自己抬出淘汰区的选手，一旦仍处于 Bottom-2，就会更容易被淘汰。

同时，Save 机制会提高“粉丝第一名被淘汰”的概率（人气冤死率在 Save 下更大），这与 Save 的规则一致：当 Bottom-2 已由综合得分确定后，最终淘汰不再看粉丝，而由评委在 Bottom-2 内做二次筛选。

## 6. 决赛专题：Rank vs Percent 的“冠军改变概率”约为 15%

在历史决赛周参赛集合固定的条件下：
- **冠军是否改变的概率**：$D_{winner}=0.1494$

解释：
- 决赛周通常参赛者更强、差距更小，因此冠军不一定像单淘汰周那样频繁翻转；
- 但 15% 仍然足够大，意味着“合成规则选择”对最终冠军并非细枝末节，尤其在粉丝/评委明显分歧的决赛。

## 7. 争议选手个案（题目点名 4 人）

下面用两类输出支撑结论：
- 决赛差异（若该选手进入决赛）：比较夺冠概率与期望名次
- 逐周风险（全部争议选手）：比较逐周淘汰概率曲线与生存汇总

### 7.1 Jerry Rice（S2）——典型“粉丝强、评委弱”的晋级
- 决赛周（S2W8）夺冠概率：Rank 0.410 vs Percent 0.732（Percent 更利于其夺冠）。
- 生存汇总（在可建模周内）：
  - Percent–Direct：存活概率 0.722
  - Rank–Direct：存活概率 0.501
  - Rank–Save：存活概率 0.000（几乎必被更早淘汰）

结论：
- Percent 对粉丝端更敏感，放大了其人气优势；
- Save 会显著压制此类“争议选手”的赛季续航。

### 7.2 Billy Ray Cyrus（S4）——多次评委垫底仍走得更远
- 生存汇总显示：四种机制下其存活概率都很低，但 Save 最“无情”。
  - Percent–Direct：0.120
  - Rank–Direct：0.041
  - Rank–Save：0.000739

结论：
- 若引入 Save，此类“评委弱势、靠粉丝维持”的选手更难积累多周晋级。

### 7.3 Bristol Palin（S11）——低评委分背景下的高名次争议
- 决赛周（S11W10）夺冠概率：Rank 0.0273 vs Percent 0.0101（两者都低，但 Rank 略提高）。
- 期望名次：Rank 2.789 vs Percent 2.894（Rank 期望名次更好一些）。
- 生存汇总（S11 可建模周）：
  - Percent–Direct：0.603
  - Rank–Direct：0.166
  - Rank–Save：0.00083

结论：
- 其争议更多体现为“赛季走多远”，而非“决赛夺冠”；
- Save 机制会大幅降低其走到后期的可能性。

### 7.4 Bobby Bones（S27）——“争议冠军”最典型案例
- 决赛周（S27W9）夺冠概率：Rank 0.167 vs Percent 0.903（Percent 极度有利）。
- 期望名次：Rank 1.843 vs Percent 1.108（Percent 期望名次显著更好）。
- 生存汇总（S27 可建模周）：
  - Percent–Direct：0.647
  - Rank–Direct：0.314
  - Rank–Save：0.042

结论（可直接写进论文）：
- 若粉丝端对某选手异常偏好而评委端持续不认可，Percent 更容易把“粉丝极化”转化为冠军；
- Rank 以及 Save 都会显著削弱这种“人气碾压技术”的夺冠路径。

## 8. 回答题面问题：哪种更偏粉丝？加入评委救人会怎样？

### 8.1 Rank vs Percent：哪种更偏向粉丝投票？
用“分歧方向”证据回答：
- 在 Rank 与 Percent 发生分歧的样本中，Rank 有 96.6% 的概率更偏向评委（judge-friendly），仅 2.9% 更偏向粉丝（fan-friendly）。

因此：
- **Percent 相对更偏向粉丝投票（更能让粉丝端在冲突时改变淘汰/冠军）。**
- **Rank 相对更偏向评委评分（在冲突时更倾向淘汰评委更不认可的人）。**

### 8.2 加入“评委从最后两名中选一人淘汰”（Save）会怎样？
- 显著降低“技术冤死”（尤其对 Percent 体系提升明显）。
- 系统性降低“争议选手”（粉丝强、评委弱）走远/夺冠的概率：Jerry Rice、Bristol Palin、Bobby Bones 的风险曲线与生存概率都显示 Save 会压制其续航。
